Description
Main Description

A context condition works as a service level agreement (SLA). If a context condition is accomplished (i.e., an SLA is violated), then an adaptation is triggered on the system to deal with the arising situation. Each context condition is represented as a triple in the form of (subject, predicate, object). A set of such triples is called a Resource Description Framework (RDF) graph. The subject (i.e., an ontology individual) denotes a resource (e.g. a Web service or a composite service operation), and the predicate expresses a relationship between the subject and the object (i.e, the value of a datatype property in the ontology).
