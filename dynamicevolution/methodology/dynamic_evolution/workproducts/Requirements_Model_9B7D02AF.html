Description
Main Description

This model is leveraged at runtime to count on the representation of the requirements that the context-aware system has to preserve at runtime. Requirements in this model have to be respected despite arising unknown context events. There are several notations for requirements modeling, including UML Use Case diagrams and goal modeling (i*, GRL, and KAOS [1]).
