Description
Main Description

The variability model describes the variants in which a system can evolve. This methodology requires a variability modeling technique to implement the variability model, such as feature modeling [1], the Common Variability Language (CVL) [2], or any domain-specific language to express variability.

For example, the variability model can be implemented with a feature model. Feature modeling offers coarse-grained variability management and it has good tool support for variability reasoning. In general terms, a system can be viewed as a set of functionalities to be preserved at runtime whose pieces are components or services. These functional pieces can be added or removed from a system at runtime in the case of new context events. To this end, systems can be abstracted as a set of features that represent variant functionalities of the system in a variability model. Therefore, evolution actions can be described in terms of the activation or deactivation of features in the variability model. The set of all currently active features represents the current configuration of the system.
