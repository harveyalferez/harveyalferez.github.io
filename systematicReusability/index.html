A. Case Study Description

This case study is about reusing Web services and composite Web services from a service composition for vacation planners in order to derive particular products. The vacation planners are intended to be used by Web and mobile applications to allow users to book flights, book hotel rooms, enjoy additional services such as car rental or booking an activity (e.g. excursions, fishing, etc.), check the vacation package costs in the users' country currency, and pay for the services by credit card. This service composition is represented by a BPMN model (see Figure 1).

    Figure 1. Vacation planner BPMN model.

In this BPMN model, the vacation planner BPMN subprocess represents the service composition, BPMN tasks are used to represent Web services (e.g. Currency Converter), and BPMN subprocesses represent composite Web services (e.g. Hotel Booking). Figure 2 shows the expanded BPMN subprocesses that represent the composite Web services in the vacation planner. They are as follows:

  1. The Flight Booking composite Web service is built in terms of its constituent Web services that are basically involved in flight booking.
  2. The Hotel Booking composite Web service contains the Web services that are involved in renting a hotel room.
  3. The Rent a Car composite Web service contains the Web services that are involved in car rental and contract an insurance policy for accidents.
  4. The Book an Activity composite Web service contains the Web services that are involved in looking for an activity (snorkeling, hiking, fishing, etc.), renting the required equipment (e.g. canoes), and booking an activity.
  5. The Credit Card Payment composite Web service contains the Web services that are involved in validating, charging, and monitoring credit card payments.

Also, the case study has a Web service that is named Currency Converter that shows the vacation package costs in the users' country currency.

2. Second Experiment

In this experiment, we replaced the BPMN inclusive gateway in Figure 1 with a BPMN exclusive gateway in order to see how this change affects the number of derived composite Web services. In this case, the Rent a Car and the Book an Activity composite Web services cannot be selected at the same time in a product. There are 120 derived versions in this experiment vs. 216 in the first experiment. As a result, we conclude that there are more derived products with more BPMN inclusive gateways than BPMN exclusive gateways.
