Tool Mentor: Tools
The set of tools that can be used to create the models and other supporting artifacts for the dynamic adapation of service compositions.
Main Description
ATLAS Model Weaver: The Weaving Model can be created with this tool. This model maps the features in the Variability Model to the elements in the Base Composition Model and Variant Models. URL: http://www.eclipse.org/gmt/amw/.

Babel: This tool can be used to create the WS-BPEL template. Babel is an open-source tool that is the result of integrating a set of techniques to translate models captured using a core subset of BPMN into WS-BPEL (Ouyang et al., 2009). The proposed techniques are capable of generating readable WS-BPEL code by discovering “patterns” in the BPMN models that can be mapped onto WS-BPEL block-structured constructs or acyclic graphs of control links. The latest version of Babel can be downloaded as an Eclipse plugin. URL: http://www.bpm.scitech.qut.edu.au/research/projects/oldprojects/babel/tools/.

Eclipse Modeling Framework (EMF): The Initial Composition Model, the Base Composition Model, and the Variant Models (which are expressed as composition models) can be created with this tool. URL: http://www.eclipse.org/modeling/emf.

FMF-Based Configuration Tool: This tool can be used to complete the WS-BPEL template. Specifically, this tool supports the manual fulfillment of the missing parts in the WS-BPEL code generated by Babel. FMF is a framework whose main goal is to facilitate the development of form-based editors that manipulate models. FMF is an extension of FEFEM. FMF is a part of the MOSKitt framework, in which the PROS has been constantly involved. A master thesis describes the FMF-based configuration tool (De La Fuente, 2012). Also, there is a video demonstration: https://vimeo.com/58989214.

MOSKitt4SPL: The Variability Model can be defined as a feature model with MOSKitt4SPL. MOSKitt4SPL has a visual Feature Model Editor to create feature models. The elements in a feature model are organized in a graphical tree. MOSKitt4SPL automatically updates the XMI representation of the feature model. Also, Context Conditions and Resolutions can be defined in MOSKitt4SPL. Finally, the Adaptation Space can be automatically generated with MOSKitt4SPL. URL: http://www.pros.upv.es/m4spl.

Protégé-OWL: The context model can be created with this tool. URL: http://protege.stanford.edu/overview/protege-owl.html.

STP BPMN Modeler: The Initial Composition Model, the Base Composition Model, and the Variant Models (which are expressed as composition models) can be created with this tool. URL: http://git.eclipse.org/c/bpmnmodeler.

Verifier: The Variability Model and its configurations (which are defined in the Adaptation Space) can be verified with this tool. The Verifier is not attached to any particular implementation. Therefore, the implementation depends on the desirable verification criteria to be applied. In our case,

1. We have chosen FAMA-FW to carry out the generic verification operations, namely void feature model, right configuration, and anomalies detection. FAMA-FW is a framework for automated analyses of feature models integrating some of the most commonly used logic representations and solvers proposed in literature (BDD, SAT, and CSP solvers).

Our MOSKitt4SPL tool supports these operations by invoking the functionality provided by FAMA-FW. The website of MOSKitt4SPL describes these operations in detail.

2. We have implemented the Semi-Aliveness and Stability verification operations with GNU Prolog. GNU Prolog is a programming language that includes a powerful constraint solver over finite domains. It uses similar algorithms as other CP solvers and the unification operation to work with Prolog facts. GNU Prolog uses very efficient algorithms based on consistency, propagation, and backtracking techniques. The non-determinism used by GNU Prolog allows to calculate several solutions for the same constraint problem and obtain the best solution (based on several criteria, such as computing time, number of backtracks, and a maximization function) that satisfies the constraints.
